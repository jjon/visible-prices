Nota Bene: Currently, there's no convenient solution for doing currency calculations in a SPARQL query. This is why we're reducing all values to pence. So if the specified PriceExpression already has a NormalizedValue, this dialog will report that fact and the graph document will not be overwritten. However, in principle, there's no reason why we couldn't record multiple NormalizedValues in a given PriceExpression, giving equivalent values in florins, guilders, or whatever. Such values would simply be of little use in a Linked Open Data context. That kind of currency calculation would have to be done server-side. Overcoming this limitation of RDF/SPARQL could be a fruitful area of continued research.
